How to publish your benchmark

The disk-duel script runs a 19-test fio-based suite on one or two drives, generates a local HTML report, and (optionally) uploads the results here.

1. Run the script

curl -fsSL https://raw.githubusercontent.com/digitalhen/disk-duel/main/disk_duel.py -o /tmp/disk_duel.py && python3 /tmp/disk_duel.py

The interactive menu auto-detects your drives. Pick one for a solo run, two to compare. Make sure fio is installed (brew install fio on macOS).

2. Upload

Pass --upload, or just say yes to the prompt at the end of the run:

python3 /tmp/disk_duel.py --upload

No API key required. The script does a small proof-of-work (~1-2 s on a recent laptop) before submitting — that's the spam protection. On success it prints the public URLs for your run and your machine.

Both URLs are stable, opaque slugs. Your hardware serial is hashed (sha256) before storage and never displayed publicly.

What's stored

  • Hardware: machine model, chip, RAM (no serial number).
  • Drive: media name (e.g. WD_BLACK SN850X 4000GB), bus protocol, internal/external flag, size, enclosure if external.
  • Per test: the primary metric (MB/s, IOPS, or p99 latency µs) plus all the bandwidth/IOPS/latency breakdowns fio emits.
  • Raw payload: the full JSON that the script wrote alongside the local HTML report, kept for future re-rendering.
