How it works.

A 23-test fio suite, an interactive drive picker, an HTML report — and one line to publish to the leaderboard.

1. Install fio

The benchmark engine itself. macOS:

Linux: sudo apt-get install fio (Debian/Ubuntu) or your distro's equivalent.

2. Run & publish

One line. Downloads the script, launches the interactive drive menu, and uploads at the end.

Don't want to upload? Drop --upload. The HTML/JSON report still lands in your current directory.

How submissions are protected

The submit endpoint is unauthenticated — the script is open source so an embedded API key would be theater. Instead, three layers run server-side, cheapest first:

  • Per-IP rate limit. 30 requests / minute, in-process.
  • Per-machine cooldown. 60 s between uploads from the same hashed serial.
  • Proof of work. The script computes a SHA-256 with 20 leading zero bits over disk-duel:v1:{serial}:{timestamp}:{nonce} before posting; the server verifies. Roughly 1–2 s on a recent laptop.

Replays of an old payload are rejected — timestamps must be fresh within ±5 minutes.

Privacy

URLs use opaque hashids slugs. Your hardware serial is hashed (sha256) on the server and never stored, returned, or rendered. Hostname is optional and not surfaced in the UI.
